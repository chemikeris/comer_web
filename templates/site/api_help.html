COMER web server API description

The COMER web server can be used from a command line API.

Submit a job using single sequence and default settings:

curl {{ request.scheme }}://{{ request.get_host }}{% url 'api_submit' %} -F sequence=TKPCQSDKDCKKFACRKPKVPKCINGFCKCVR

After job is submitted, the server returns job identifier (jobID) that can be used to query job status:

curl {{ request.scheme }}://{{ request.get_host }}{% url 'api_job_status' job_id='JobID' %}

When job finishes, its results can be retrieved in JSON format:

curl {{ request.scheme }}://{{ request.get_host }}{% url 'api_results_json' job_id='JobID' %}

The full results of a finished job can be also retrieved as a zip file:

curl {{ request.scheme }}://{{ request.get_host }}{% url 'api_results_zip' job_id='JobID' %}

If necessary, job input data, search options or error log can be queried as well:

curl {{ request.scheme }}://{{ request.get_host }}{% url 'api_job_input' job_id='JobID' %}

curl {{ request.scheme }}://{{ request.get_host }}{% url 'api_job_options' job_id='JobID' %}

curl {{ request.scheme }}://{{ request.get_host }}{% url 'api_job_error' job_id='JobID' %}

A demo Python script that submits a job and waits for its results using this COMER is available.
